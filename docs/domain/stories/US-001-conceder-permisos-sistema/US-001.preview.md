# Preview de Interfaz — HU #US-001: Conceder los permisos del sistema

> ⚠️ **PROPUESTA PENDIENTE DE VALIDACIÓN CON DISEÑO** — Este prototipo debe ser revisado y aprobado por el equipo de diseño antes de implementarse.
>
> Formato: **Mermaid (flujo de navegación)** · Plataforma inferida: **mobile** · Línea gráfica: *defaults* (sin proyecto frontend en el workspace).

```mermaid
flowchart TD
    %% Generado para HU #US-001: Conceder los permisos del sistema

    Start(["El Oyente abre Sonus por primera vez"]) --> ChkOnb{"¿onboardingCompleted<br/>= false?"}
    ChkOnb -->|"No (ya completado)"| Skip["Arranque normal<br/>(fuera de esta HU)"]
    ChkOnb -->|"Sí (primera ejecución)"| ChkApi{"¿Android 13+<br/>(API >= 33)?"}

    %% Rama API < 33 — Escenario 3
    ChkApi -->|"No (API < 33)"| SkipStep["Omitir paso de permiso<br/>(notificación implícita)"]
    SkipStep --> Folders["Pantalla: Selección de<br/>Carpetas Fuente (US-002)"]

    %% Rama API 33+ — idempotencia (Escenario 5)
    ChkApi -->|"Sí (API 33+)"| ChkGranted{"¿POST_NOTIFICATIONS<br/>ya concedido?"}
    ChkGranted -->|"Sí"| Folders

    %% Pantalla de rationale — Escenario 1
    ChkGranted -->|"No"| Rationale["Pantalla: Rationale de Notificaciones<br/>· Título + explicación del beneficio<br/>· Ícono<br/>· [Permitir notificaciones] (primario)<br/>· [Omitir por ahora] (secundario)"]

    Rationale -->|"toca 'Omitir por ahora'"| Folders
    Rationale -->|"toca 'Permitir notificaciones'"| CanShow{"¿El SO puede mostrar<br/>el diálogo?<br/>(no 'no volver a preguntar')"}

    %% Diálogo del SO — Escenarios 1 y 2
    CanShow -->|"Sí"| SysDialog(["Diálogo nativo del SO:<br/>¿Permitir notificaciones?"])
    SysDialog -->|"Concede"| Granted["Registrar permiso concedido<br/>(habilita notificación persistente<br/>del Foreground Service)"]
    Granted --> Folders
    SysDialog -->|"Niega"| DenyToast["Aviso no intrusivo:<br/>'Podrás activarlas luego en Ajustes'"]
    DenyToast --> Folders

    %% Negación permanente — Escenario 4
    CanShow -->|"No (negación permanente)"| PermMsg["Mensaje: habilitar manualmente<br/>· [Abrir Ajustes] (enlace al SO)<br/>· [Continuar]"]
    PermMsg -->|"toca 'Abrir Ajustes'"| SysSettings(["Ajustes de notificaciones<br/>del sistema (fuera de Sonus)"])
    PermMsg -->|"toca 'Continuar'"| Folders
    SysSettings -.->|"el Oyente regresa a Sonus"| Folders
```

## Leyenda de trazabilidad (AC → flujo)

| AC | Rama del diagrama |
|----|-------------------|
| Escenario 1 (Flujo Principal) | Rationale → diálogo SO → Concede → Carpetas Fuente |
| Escenario 2 (Negación simple) | Diálogo SO → Niega → aviso → Carpetas Fuente |
| Escenario 3 (API < 33) | Omitir paso → Carpetas Fuente |
| Escenario 4 (Negación permanente) | ¿SO puede mostrar? = No → mensaje + Abrir Ajustes |
| Escenario 5 (Idempotencia) | ¿Ya concedido? = Sí → Carpetas Fuente |
| Escenario 6 (Autarquía) | Invariante transversal (sin nodo de red; verificable en manifiesto) |
